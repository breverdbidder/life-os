# Naming Best Practices Reference

## Trademark Filing Templates

### USPTO TEAS Plus Application Template

```
MARK: [NAME]
OWNER: [Individual/Company Name]
ADDRESS: [Full Address]

CLASS 9 - Computer Software
Description: Downloadable computer software using artificial intelligence 
for [domain-specific function]; [additional features]

First Use in Commerce: [Date]
Specimen: Screenshot of software showing "[MARK]" branding

CLASS 42 - SaaS Services  
Description: Software as a service (SAAS) featuring software for 
[domain-specific function]; [additional features]

First Use in Commerce: [Date]
Specimen: Website screenshot of SaaS login/subscription page

Filing Basis: 1(a) Use in Commerce OR 1(b) Intent to Use
Filing Fee: $350 per class
```

### Class Selection Guide

| Class | Type | Use When |
|-------|------|----------|
| 9 | Goods (Software) | Downloadable apps, desktop software |
| 41 | Services (Education) | Training, certification programs, courses |
| 42 | Services (SaaS/Tech) | Cloud software, consulting, technical services |

## Brand Architecture Patterns

### Pattern 1: Parent → Product
```
EVEREST CAPITAL USA (Parent Entity)
    └── BidDeed.AI (Product)
        └── Powered by The Shapira Formula™
```
**Use when:** Existing company launching new product line

### Pattern 2: Methodology → Implementation
```
THE SHAPIRA FORMULA™ (Methodology - Ariel owns personally)
    └── BidDeed.AI (Software - Company licenses)
```
**Use when:** IP ownership separation critical for exit strategy

### Pattern 3: Framework → Stages → Products
```
THE EVEREST ASCENT™ (12-Stage Framework)
    ├── LienLogic™ (Stage 4)
    ├── BidScore™ (Stage 7)
    └── BidDeed.AI (Complete Platform)
```
**Use when:** Multiple product lines from single methodology

### Pattern 4: Triple-Layer
```
EVEREST CAPITAL USA (Entity)
    └── THE EVEREST ASCENT™ (Framework)
        └── The Shapira Formula™ (Metric/Calculation)
            └── BidDeed.AI (Software Product)
```
**Use when:** Maximum IP protection + brand equity preservation

## Investor Pitch Language

### For Methodology Names
> "You're not investing in software—you're licensing [METHODOLOGY NAME]. 
> It's the result of [X years] of [domain expertise]. [PRODUCT NAME] is 
> the only authorized platform to execute it."

**Example:**
> "You're not investing in software—you're licensing The Shapira Formula™. 
> It's the result of 35 years of foreclosure auction expertise. BidDeed.AI 
> is the only authorized platform to execute it."

### For Personal IP Ownership
> "[FOUNDER NAME] owns [METHODOLOGY] personally. [COMPANY] has an exclusive 
> license. Investors own equity in the LLC, not the IP. This protects 
> [METHODOLOGY] through any corporate event—acquisition, bankruptcy, or pivot."

### For Exit Strategy
> "At acquisition, [METHODOLOGY] is negotiated separately from equity. 
> Similar to a chef's recipe IP in a restaurant sale—it's worth 2-5x 
> the business valuation alone."

## Exit Strategy Considerations

### IP Licensing Models

**Model 1: Exclusive Perpetual License**
- Company has exclusive rights forever
- IP owner retains ownership
- Best for: Family businesses, lifestyle companies

**Model 2: Exclusive with Reversion**
- Company has exclusive rights while founder involved
- Reverts to non-exclusive if founder departs
- Best for: Founder-dependent expertise

**Model 3: Separate IP Sale**
- IP negotiated separately at exit
- 2-5x multiplier on methodology value
- Best for: Maximizing exit value

### Valuation Multipliers

| Asset Type | Typical Multiplier | Example |
|------------|-------------------|---------|
| Software Code | 2-3x revenue | $1M ARR = $2-3M valuation |
| Methodology IP | 5-10x internal alpha | $300K/year alpha = $1.5-3M valuation |
| Personal Brand | 10-20x annual consulting fees | $100K/year = $1-2M premium |

**Combined Example:**
- Software: $500K ARR × 3 = $1.5M
- Methodology: $300K alpha × 7 = $2.1M  
- Personal Brand: $100K × 15 = $1.5M
- **Total Valuation: $5.1M**

## Common Naming Mistakes

❌ **Too Generic**: "Real Estate Analytics" → No trademark protection  
✅ **Better**: "Shapira Real Estate Analytics" → Personal brand + function

❌ **Unclear Domain**: "Summit Ascent Solutions" → What industry?  
✅ **Better**: "Summit Foreclosure Intelligence" → Clear vertical

❌ **No AI Signal**: "Property Evaluator" → Sounds manual  
✅ **Better**: "Property AI Evaluator" → Tech-forward

❌ **Poor Pronunciation**: "Xzyloq" → Hard to spell/say  
✅ **Better**: "BidLogic" → Phonetic, memorable

❌ **Geographic Lock-in**: "Brevard Foreclosure Tool" → Can't scale  
✅ **Better**: "Foreclosure Intelligence Platform" → Scalable

## Name Testing Checklist

Before finalizing a name, verify:

- [ ] USPTO Classes 9+42 cleared (LOW risk)
- [ ] .com OR .ai domain available
- [ ] Pronounceable in one try
- [ ] No negative connotations in major languages
- [ ] Scalable beyond initial market
- [ ] Works in brand hierarchy
- [ ] Founder comfortable with personal association (if founder-based)
- [ ] Passes "business card test" (looks professional printed)
- [ ] Passes "phone test" (easy to spell when heard)
- [ ] Passes "investor pitch test" (creates clear narrative)
